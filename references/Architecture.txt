Reading the diagram top to bottom:
Every request enters through Cloudflare first — nothing ever hits Vercel directly. Cloudflare absorbs DDoS, fires WAF rules, and hides your origin server IP. Vercel handles the React frontend and serverless API routes. Those routes talk to three services — Supabase for data, Cloudinary for images, and Razorpay for payments.
The blue layer at the bottom is entirely yours to build and own. That's where your cybersecurity background turns this from a basic website into something properly hardened.

The most important data flows to understand:
When a user places an order — browser hits Cloudflare → Vercel API route → creates order in Supabase → calls Razorpay → Razorpay fires a webhook back to your Vercel API confirming payment → you update order status in Supabase. Payment data never touches your database directly.
When your friend uploads a product photo — it goes through a Vercel serverless function that strips EXIF data first, then pushes the clean image to Cloudinary. The Supabase product record stores only the Cloudinary image URL.

Set up the project scaffold — Vite + React, connect to Supabase, configure environment variables
Build the security headers and CSP from day one, not as an afterthought
Set up Supabase tables with RLS enabled before writing a single frontend component
Then build the UI
